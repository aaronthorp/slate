    Introduction

    Welcome to the PartsMatch API! You can use our API to access various PartsMatch API endpoints, allowing you to integrate third party products with the PartsMatch system.

    Our API is JSON based, you can view code examples and responses in the dark area to the right, and you can switch the programming language of the examples with the tabs in the top right where available.

    If examples are not available for a specific endpoint on the right hand side, this endpoint may not currently be available. If an endpoint is unavailable for any reason you will see an error response.

    Making Calls

    Developer Token

    Once you have been issued a PartsMatch Developer Token, you can make calls to the API. You will need to include your developer token in the header of all requests to the API in the following format:

    PartsMatch: {devToken}

    Headers

    Default headers:

    Content-type: application/json
Content-length: {length}
PartsMatch: {devToken}

    The API communicates in JSON data, please ensure the following headers are used for all calls to ensure there are no issues with communications. For some calls you may also need to include the Authorization header after logging in to access user specific information.

    Authorization Token

    PartsMatch uses API tokens allow access to the API at a user level. You can request a token for a user by sending the username and password to the endpoint in the example. All API requets to the server require this token to be passed in the header in the following format:

    Authorization: Bearer {token}

    You must replace {token} with the API key returned from a "User Login" call.

    Authentication

    User Login

    To authorize, use this code:

    {
    "username": "user@partsmatch.com.au",
    "password": "password123"
}

    The above command returns JSON structured like this:

    {
  "token": "string",
  "tokenExpires": "ISO encoded date string",
  "id": "user id"
}

    The above command returns JSON error response like this:

    {
  "error": "error-code",
  "reason": "Human readable error string"
}

    Provide the users username and password to this endpoint to retrieve a login token for the user to use in all user-specific endpoints.

    HTTP Request

    POST https://api.partsmatch.com.au/users/login

    Enquiries

    PartsMatch Member's create Enquiries to source and search for parts. The Enquiries API allows 3rd party applications to submit enquiries on the members behalf.

    The Enquiries API is only available to a Member user. Calling this API as a Supplier will yield an error response.

    Submit New Enquiry

    To submit an enquiry, use this code:

    {
    "referenceNo": "ABC123",
    "notes": "Additional Notes Here",
    "photos": [
        {"url": "", "title": ""}
    ],
    "vehicle": {
        "year": "2017",
        "make": "Mazda",
        "model": "CX-5",
        "badge": "",
        "trans": "",
        "engine": "",
        "odometer": "",
        "colour": "Red",
        "vin": "JM00324734238974234"
    },
    "items": [
        {
            "description": "L/H/F Door",
            "partNumber": "AW123123123",
            "listPrice": 150.43,
            "quantity": 1
        },
        {
            "description": "Bonnet",
            "partNumber": "AW432432432",
            "listPrice": 440.29,
            "quantity": 1
        }
    ]
}

    The above command returns JSON structured like this:

    {
  "success": true,
  "url": "https://app.partsmatch.com.au/estimates/view/KJ35KLqwQnMSabfew",
  "id": "KJ35KLqwQnMSabfew"
}

    The above command returns JSON error response like this:

    {
  "success": false,
  "errors": [
    ...
    "Duplicate Reference Number",
    "Required Field: {field}",
    ...
  ]
}

    This generates a new Enquiry on behalf of the PartsMatch user.

    HTTP Request

    POST https://api.partsmatch.com.au/api/enquiries

    On response you can direct the user to the url parameter in thier browser to complete the PartsMatch enquiry.

    Get Enquiry Status

    The above command returns JSON structured like this:

    {
    "referenceNo": "123",
    "items": [
        {
            "description": "L/H/F Door",
            "partNumber": "AW123123123",
            "referenceNo": "",
            "quotedPrice": 100.82,
            "supplier": "Some Cheap Supplier",
            "ordered": false,
            "quoted": true,
            "orderId": null,
            "orderDate": null,
            "receivedDate": null,
            "expectedDate": null,
            "orderTracking": null,
        },
        {
            "description": "Bonnet",
            "partNumber": "AW432432432",
            "referenceNo": "",
            "quotedPrice": 375.39,
            "supplier": "Another Cheap Supplier",
            "ordered": true,
            "quoted": true,
            "orderId": "Ax338fkDASFsdfDFE",
            "orderDate": "2018-06-29 11:34:00",
            "receivedDate": null,
            "expectedDate": "ASAP",
            "orderTracking": "Courier AM",
        }
    ]
}

    This endpoint retrieves the current status of an enquiry and its items, including current selected item supplier, pricing and order status details.

    You can specify the enquiryId from the New Enquiry endpoint, or the users internal referenceNo that was used when the enquiry was created to retrieve details from this endpoint

    HTTP Request

    GET https://api.partsmatch.com.au/api/enquiries/<ID>

    URL Parameters

    Parameter Description
    ID The enquiryId or referenceNo of the Enquiry to retrieve (estimateId can be found in the New Enquiry endpoint)

    Quotes

    List Quotes

    Retrieve a list of quotes for a user.

    HTTP Request

    GET https://api.partsmatch.com.au/api/quotes

    Get Quote

    Retrieve details of a quote for a user.

    HTTP Request

    GET https://api.partsmatch.com.au/api/quotes/<ID>

    URL Parameters

    Parameter Description
    ID The quoteId of the Quote to retrieve

    Create Quote Response

    Create a response to a specific quote for a user.

    HTTP Request

    POST https://api.partsmatch.com.au/api/quotes/<ID>

    URL Parameters

    Parameter Description
    ID The quoteId of the Quote to retrieve

    Cancel Quote Response

    Cancel a previously created quote response.

    HTTP Request

    DELETE https://api.partsmatch.com.au/api/quotes/<ID>

    URL Parameters

    Parameter Description
    ID The enquiryId or referenceNo of the Enquiry to retrieve (estimateId can be found in the New Enquiry endpoint)

    Webhooks

    All webhooks are required to return a 200 response to flag as successfully received.

    In the case of an endpoint failure, webooks will retry upto 10 times over a period of 24 hours. If the endpoint is still failing, it will be disabled and developer notified. Webhooks that are disabled will need to be re-created.
